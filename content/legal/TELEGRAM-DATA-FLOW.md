# Карта обработки данных в Telegram

Дата технической сверки: 24 июля 2026 г.  
Контур: `@academic_saloon_bot`, личные чаты клиентов, закрытая рабочая
Telegram-группа с темами заказов и канал `@akademsalon`.

Это внутренняя карта фактического движения данных. Она не подменяет
уведомление Роскомнадзора и отдельную правовую оценку трансграничной передачи.

## 1. Что приходит из Telegram

| Данные | Источник | Для чего | Где сохраняются |
|---|---|---|---|
| Telegram ID | Bot API | вход, привязка заказа, доставка сервисных уведомлений | `users.id`, `orders.user_id`, журналы доставки |
| username, отображаемое имя | профиль Telegram | обращение к клиенту и поиск его дела мастером | `users.username`, `first_name`, `last_name` |
| текст сообщений | личный чат клиента или рабочая тема | согласование и исполнение заказа | `messages.text`, Telegram-чат |
| файлы, фото, голосовые | добровольная отправка клиентом/мастером | исполнение позиции и переписка | Telegram `file_id`, имя/размер и служебная запись в SQLite |
| callback-команды и deep-link | кнопки Бота | выбранное действие, вход, привязка заказа | результат действия и необходимые события в SQLite |
| согласие/отказ от рекламы | `/startnews`, `/stopnews` | доказательство opt-in/opt-out | `users.subscribed`, `subscribed_at` |
| запрос удаления | `/delete_me` + подтверждение | отзыв и реализация прав субъекта | немедленная операция удаления/обезличивания и служебное уведомление владельцу |

Бот не запрашивает паспорт, медицинские сведения, биометрию, сведения о
третьих лицах или специальные категории данных. Интерфейс и Политика должны
предупреждать не присылать их добровольно.

## 2. Что уходит в Telegram

| Получатель | Содержание | Ограничение |
|---|---|---|
| клиент | нейтральные статусы заказа, сообщения мастера, согласованные файлы, фирменное подтверждение платежа | официальный чек НПД не подменяется внутренним PDF |
| закрытая рабочая группа | карточка дела, согласованные параметры, переписка и файлы, необходимые мастеру | доступ только рабочим аккаунтам; публичный канал не используется |
| владелец/администратор | алерты оплаты, ошибок, удаления и действий, требующих решения | без платёжных секретов и токенов доступа |
| публичный канал | только отдельно подготовленные публикации мастерской | данные заказа и клиента автоматически не публикуются |

## 3. Связи систем

```text
клиент в Telegram
    ↕ Bot API
бот/API на VPS в РФ
    ↔ SQLite на том же VPS
    ↔ закрытая рабочая Telegram-группа
    ↔ сайт/кабинет через собственный API
```

Telegram самостоятельно хранит экземпляры переписки и файлов на своей
платформе. В SQLite хранятся необходимые метаданные и `file_id`; при скачивании
сайт запрашивает файл у Bot API только после проверки доступа к заказу.

## 4. Основания и минимизация

- Заявка, переписка, статусы и выдача: заключение и исполнение договора,
  инициированного клиентом.
- Налоговые и платёжные сведения: исполнение обязанности по закону.
- Реклама: только отдельная команда `/startnews`; `/stopnews` выключает её,
  не отключая сервисные сообщения.
- Публикация отзыва/скриншота: отдельное согласие по статье 10.1; обычная
  переписка и оплата такого согласия не дают.
- В аналитику сайта Telegram ID, текст, тема заказа, файлы и токены не
  передаются.

## 5. Удаление и ограничения

`/delete_me` удаляет локальный профиль, сессии, маркетинговую отметку и
бонусный счёт, отключает гостевые ссылки и отвязывает Telegram ID от заказов.
Договорные и налоговые записи, которые закон требует сохранить, не удаляются
по одному клику и используются только для соответствующей обязанности.

Команда не может сама гарантировать удаление копий сообщений и файлов на
стороне Telegram, в рабочих чатах и резервных копиях. Для полного запроса
проводится ручная проверка через `support@akademsalon.ru`; удаление в доступных
системах документируется, а исключения по закону блокируются.

## 6. Обязательные организационные действия владельца

1. Проверить и при необходимости актуализировать уведомление оператора
   персональных данных.
2. До продолжения передачи через Telegram получить документированный вывод по
   статье 12 152-ФЗ: получатель, страны, цель, объём и требуемое уведомление.
3. Зафиксировать перечень рабочих аккаунтов с доступом к закрытой группе и
   пересматривать его при каждом изменении команды.
4. Утвердить срок удаления рабочих файлов и ротации резервных копий.
5. Не использовать публичный канал для клиентских материалов без отдельного
   согласия на конкретные сведения и площадку.
