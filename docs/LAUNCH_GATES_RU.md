# Контроль запуска

Дата: 24 июля 2026 года

Этот список не заменяет заключение российского юриста, бухгалтера и специалиста
по персональным данным. Он не позволяет случайно включить трафик раньше, чем
публичное обещание и фактическая работа станут одним продуктом.

## P0 — блокирует публичный запуск

- [ ] Владелец подтвердил фактический статус бизнеса: НПД, ИП или ООО.
- [ ] Бухгалтер проверил лимит и ограничения НПД либо режим ИП/ООО, НДС 2026,
      эквайринг, ККТ и выдачу чеков.
- [ ] В оферте, политике и интерфейсе указан настоящий адрес для
      корреспонденции; заглушки удалены.
- [ ] Российский юрист проверил итоговую фактическую модель, оферту,
      спецификацию, возвраты и рекламу.
- [ ] Проверена необходимость уведомления Роскомнадзора; уведомление подано
      либо сохранено обоснование исключения.
- [ ] Первичный сбор и хранение персональных данных граждан РФ локализованы в
      российской базе.
- [ ] С хостингом, экспертами, CRM, платёжным оператором и иными обработчиками
      оформлены необходимые поручения и условия конфиденциальности.
- [ ] Создан реестр целей, данных, оснований, обработчиков и сроков удаления.
- [ ] Основной заказ, файлы и оплата доступны через портал; Telegram не является
      единственным хранилищем полного заказа.
- [ ] Продукт фактически не включает готовые аттестационные работы, главы,
      экзамены, вход в LMS, вымышленные данные или обход проверки.
- [ ] Все старые рекламные объявления, посты, закрепы, описания бота и
      автосообщения приведены к новой фактической модели.
- [ ] Запрос «сделайте за меня» переводится в допустимый формат или отклоняется.
- [ ] До оплаты сохраняются номер и снимок спецификации, версия оферты и время
      отдельных согласий.
- [ ] Дополнительные услуги не отмечены заранее и подтверждаются отдельно.
- [ ] Публикация отзыва технически невозможна без сохранённого отдельного
      согласия.
- [ ] Цена и состав продукта совпадают на сайте, в API, боте, админке,
      спецификации и платёжной форме.
- [ ] Создана проверенная резервная копия базы; есть инструкция восстановления.
- [ ] Токены не передаются в URL, у сессии есть срок жизни и отзыв.
- [ ] Секреты отсутствуют в Git; production-конфигурация проверена отдельно.

## P1 — блокирует активное масштабирование

- [ ] Главная объясняет продукт за 5–10 секунд на телефоне и компьютере.
- [ ] Короткая анкета задаёт этап и задачу, а не заставляет выбирать внутренний
      юридический контур.
- [ ] Перед оплатой клиент видит простое резюме результата рядом с полной
      спецификацией.
- [ ] В кабинете есть следующий шаг, дедлайн, история версий и решений.
- [ ] Подготовлены реальные обезличенные образцы диагностики, redline и
      консультационного результата.
- [ ] У каждого специалиста есть договор, NDA, границы доступа и условия прав.
- [ ] Реализованы удаление аккаунта и файлов, отзыв согласия и выгрузка данных.
- [ ] Настроены процедура инцидента и ответственные лица.
- [ ] Аналитика включается только после управляемого необязательного согласия.
- [ ] Выполнены мобильные, desktop, keyboard, screen-reader и reduced-motion
      проверки ключевых путей.
- [ ] Выполнены тестовые заказ, спецификация, оплата, чек, этап, возврат,
      удаление данных и согласие на отзыв.

## P2 — рост после безопасного запуска

- [ ] Создан редакционный план по экспертным поисковым кластерам.
- [ ] Старые URL переписаны без дорвеев, настроены canonical и sitemap.
- [ ] Реклама и платные Telegram-размещения проходят проверку маркировки,
      рекламодателя, ERID и передачи в ОРД/ЕРИР.
- [ ] Реальные отзывы публикуются с доказуемой датой и объёмом согласия.
- [ ] A/B-тесты меняют только один элемент и используют защитные метрики.
- [ ] Утверждены недельный и месячный отчёты по воронке, качеству и рискам.
