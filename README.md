# Академический Салон

Сайт мастерской консультационной, учебно-методической, редакторской и авторской помощи.
Контур А охватывает работу с собственным материалом клиента; Контур Б — авторский заказ
для прямо указанной цели вне учебной и научной аттестации. Статичная публичная часть
реализована на HTML + CSS + ванильном JS без сборщика.

Дизайн-направление — **«Оттиск»** (университетская пресса): тёплая бумага, антиква Literata,
гротеск Golos Text, моноширинные цифры JetBrains Mono, тонкие линейки, строки с отточиями,
сургучный акцент, SVG-печати. Принцип движения: анимируется только то, что «печатается,
пишется или считается».

## Страницы
- `index.html` — главная: три законных маршрута, процесс, каталог, гарантии и FAQ
- `configurator.html` — многопозиционный бланк заявки с отдельными предметами, целями и входными данными
- `tariffs.html` — прайс-каталог консультаций, диагностики, редактуры, оформления и сопровождения
- `avtorskiy-zakaz.html` — авторские тексты вне аттестации с фиксацией назначения, автора и режима прав
- `specifikaciya.html` — описание индивидуальной части договора и заполненные PDF-образцы
- `guarantees.html` — «Устав мастерской»: измеримые обязательства, подтверждения и самопроверка
- `check.html` — бесплатная проверка текста на канцелярит и машинальность (анализ в браузере, текст никуда не уходит)
- `zayavka.html` — дело заказа: позиции, график, документы и платёжный контекст
- `dashboard.html` — личный кабинет клиента, синхронизированный с API и Telegram
- `admin.html` — закрытый кабинет мастера
- `404.html` — страница ошибки
- `oferta.html`, `privacy.html`, `terms.html`, `academic-integrity.html`, `refunds.html`, `requisites.html` — связанный правовой комплект

## Структура
- `assets/css/styles.css` — дизайн-система «Оттиск» (токены и компоненты: sheet, dotrow, ledger, paysteps, report-sheet, seal, tag…)
- `assets/js/app.js` — общий слой: ценовая логика, каталог законных услуг, UX-утилиты и общий chrome
- `assets/js/cart.js` — многопозиционная заявка и структурированный payload по каждой строке
- `backend/` — воспроизводимый Telegram-бот и same-origin API без базы, токенов и клиентских файлов
- `backend/catalog/pricing.v1.json` — канонический versioned-каталог результатов и цен
- `scripts/site-audit.mjs` — структурный, SEO- и safety-аудит публичных страниц
- `scripts/build-public.mjs` — allow-list сборка публичной статики в `dist/`
- `scripts/generate-spec-v2.py` — канонический генератор заполненных PDF-образцов Спецификации
- `content/legal/` — схема Спецификации, матрица персональных данных и эксплуатационные legal-checklists
- `assets/img/` — favicon, OG-обложка, apple-touch-icon
- `robots.txt`, `sitemap.xml` — SEO

## Клиентские черновики

localStorage хранит только клиентский черновик и демонстрационное состояние. Он не является
договорным снимком. Принятая Спецификация должна формироваться и неизменно храниться сервером
с номером редакции, построчными условиями и контрольными суммами.

## Локальный запуск

```bash
python3 -m http.server 8000     # http://localhost:8000
```

Проверка и безопасная публичная сборка:

```bash
npm test
npm run build
```

Backend запускается и тестируется отдельно по инструкции
[`backend/README.md`](backend/README.md). Его `.env`, рабочая SQLite-база,
Telegram-файлы, логи и резервные копии не должны попадать в Git или `dist/`.

Перед просмотром только фронтенда API-запросы на localhost ожидаемо обращаются
к production API. Для end-to-end разработки используйте локальный reverse
proxy и тестовое окружение, а не рабочую базу.

## Деплой

Production обслуживает Nginx на настроенном VPS; корень публичной статики —
`/var/www/academic_saloon/dist`. Публикуется только результат `npm run build`
из проверенного релизного коммита. Сборщик исключает закрытые исходники,
backend, локальные отзывы без доказанного согласия, тесты и рабочие артефакты.

Публикация выполняется адресной синхронизацией через `rsync` по отдельному
SSH-ключу деплоя. Ключи и реквизиты доступа в репозиторий не добавляются.
Перед выпуском обязательны серверная резервная копия, `quick_check` базы,
миграции на копии и smoke-тесты по
[`backend/README.md`](backend/README.md). Статику и backend переключают как
один согласованный релиз: версии согласия и цен не должны расходиться.

Локальный `.github/workflows/deploy.yml` является заготовкой для GitHub Actions и не
считается активным, пока workflow не добавлен в ветку и в GitHub Secrets не настроены
`SSH_HOST`, `SSH_USER`, `SSH_KEY`, `DEPLOY_PATH` и при необходимости `SSH_PORT`.

## Особенности реализации
- Контент виден без JS; шапка/подвал инжектируются прогрессивно; `prefers-reduced-motion` уважается везде.
- View Transitions API для переходов между страницами (с фолбэком), speculation rules / prefetch для мгновенной навигации.
- Шрифты — Google Fonts с кириллицей: Literata (variable), Golos Text, JetBrains Mono.
- Сервер и Telegram-бот входят в `backend/` в обезличенном виде. Публичный
  frontend, каталог цен, редакции согласий, бот и API проходят совместные
  contract-тесты; клиентский черновик не выдаётся за договорный снимок.
